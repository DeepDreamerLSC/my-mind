# 风险清单

## 当前风险

- 工作区变更多：功能、配置、设计、测试和 artifact 混在一起，容易提交时丢失边界。
- 质量门禁不足：PDF 题目裁切一旦漏题、错切或吞并答案解析，会直接影响教育场景可信度。
- 环境配置泄露：`.env.*`、生产配置、日志和本地 artifact 需要严格区分可提交与不可提交内容。
- 前后端契约漂移：前端在线排版、文件上传下载和 OpenAPI 可能与后端资源模型不同步。
- 主仓库与生产副本漂移：`../edu-agent` 与 `../prod/edu-agent` 的分支、配置和部署状态需要定期核对。

## 风险处理原则

- 先把风险写成可复核的门禁或测试，再推动代码收敛。
- 生产配置只记录字段和决策，不保存真实密钥。
- 大体积样例和日志只保留索引，不默认纳入知识库正文。
