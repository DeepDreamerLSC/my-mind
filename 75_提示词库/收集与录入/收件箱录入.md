# 收件箱录入

## 用途

把用户随手丢来的链接或片段先安全写入 `00_收件箱/`，等待后续整理。

## 当前能力

- 支持一个或多个链接。
- 优先用 `yt-dlp` 抓取视频或网页基础信息。
- 记录标题、作者或频道、作者编号、内容编号、发布时间、时长、平台、简介或文案摘录、标签、分类、缩略图、互动数据和字幕可用性。
- 对 YouTube 链接，会尝试从公开视频页发现官方转录链接，并记录转录来源、章节目录、片段数和估算词数。
- 如果 `yt-dlp` 提供字幕轨道，会解析字幕来源、语言、格式、片段数和估算词数。
- 对抖音公开分享链接，会解析短链后的移动端公开页，尽量收集作品文案、作者、发布时间、时长、封面、作品编号、点赞、评论、收藏和分享数。
- 对小红书公开分享链接，会解析短链后的公开笔记页，尽量收集笔记标题、正文文案、作者、发布时间、封面、笔记编号、图片数、点赞、评论、收藏和分享数。
- 显式要求内容摘录时，可通过 `--extract-content` 对公开视频抽音频、转写，统一转成简体中文，并用术语表修正常见英文产品名误识别，再保存摘要、关键点、完整时间线摘录和完整转写摘录。
- 抓取失败时仍然生成收件箱笔记，并记录失败原因。

## 使用方式

```text
收件箱录入：
https://www.youtube.com/watch?v=...
```

Codex 应调用：

```bash
python3 .codex/skills/inbox-capture/scripts/capture_link.py "链接"
```

如果需要视频内容摘录，Codex 可调用：

```bash
python3 .codex/skills/inbox-capture/scripts/capture_link.py --extract-content --transcribe-backend faster-whisper --transcribe-model tiny "链接"
```

## 处理边界

- 只入箱，不晋升为长期知识。
- 普通入箱不下载视频或音频。
- 普通入箱不做 Whisper 转写；只有显式要求内容摘录时才转写。
- 不使用登录态或 cookie。
- 不绕过平台访问控制，不抓取评论区，不下载媒体文件。
- 不把超长版权转录全文复制进仓库，只保留可追溯来源和结构化解析结果。
- 不默认复制字幕原文或歌词。
- 显式内容摘录模式不再截断时间线段落和转写摘录，适合需要认真分拣的视频。
- 如果 ASR 把 Codex、OpenAI、Skill、Prompt、PPT 等术语识别成中文谐音，优先扩展脚本里的术语表，而不是只修单篇笔记。
- `tiny` 模型只适合快速验证，中文错字较多；要晋升长期资料时建议用 `small` 或 `medium` 重跑。
