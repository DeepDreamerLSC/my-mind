# 我的心智

我的心智是一个个人数据资产系统，目标是把资料、人工智能对话、行业信号、项目记录、原子笔记、主动回忆、洞察和复盘组织成一个可长期运行的数据飞轮。

## 当前阶段

- 阶段一：最小结构
- 阶段二：处理当前设计讨论
- 阶段三：收件箱巡检自动化与前台协作试运行

## 可用技能

- `inbox-capture`：把视频链接、网页链接或粗略片段写入 `00_收件箱/`。第一版会收集标题、作者或频道、作者编号、内容编号、发布时间、时长、平台、简介或文案摘录、标签、分类、缩略图、互动数据和字幕可用性；YouTube 链接会尝试解析字幕轨道的来源、语言、格式、片段数和估算词数，也会尝试发现官方转录来源并提取章节目录；抖音和小红书公开分享链接会走公开页面兜底解析，尽量收集文案、封面和互动数据；公开视频暴露媒体/音频地址且未超过时长上限时默认抽音频转写，统一转为简体中文，并用术语表修正常见英文产品名误识别，再保存摘要、关键点、完整时间线摘录和完整转写摘录；解析失败也会保留链接和失败原因。
- `inbox-triage`：扫描 `00_收件箱/` 中 `处理状态: 待分拣` 的笔记，生成中文分拣巡检报告；启用自动分拣时可把来源回写为 `已分拣` 并追加 `分拣记录`，同时刷新 `05_流转区/` 的待读、待沉淀和待核验队列，提醒用户先读，并把读后反馈回写到 `阅读思考` 后再决定是否继续沉淀。
- `inbox-distill`：把单条收件箱笔记转成候选沉淀物；第一版支持生成 `75_提示词库/` 候选提示词，并把来源里的 `阅读思考` 纳入候选输入；在 `--write` 时把来源状态回写为 `已处理`，表示用户已确认继续沉淀，但仍未自动晋升长期知识。
- `frontier-watch`：从稳定 RSS/Atom 来源巡检 `AI与Agent工具`、`工作流与知识系统`、`商业`、`管理` 四类前沿情报，默认只选最近时间窗口内且有发布时间的项目，并把入选项外文摘要译写为简体中文，生成 `85_运行记录/前沿情报巡检-*.md`；只有显式 `--write-inbox` 才把入选项写入 `00_收件箱/`。

## 已启用自动化

- `收件箱待分拣巡检`：Codex App `cron` 自动化，当前每 3 小时巡检一次 `00_收件箱/`；只做低风险自动分拣和阅读队列整理，不在用户未确认前生成资料库、原子笔记、提示词库或项目草稿。

## 第一版目录

- `00_收件箱/`：未处理内容入口
- `05_流转区/`：收件箱到长期知识之间的短期行动队列，包括待读、待沉淀、待核验和暂缓
- `10_项目/`：项目上下文、决策、任务、问题和风险
- `20_资料库/`：原始资料和初步整理资料
- `30_原子笔记/`：长期可复用知识点
- `35_主动回忆/`：回忆卡片和复习队列
- `60_行业情报/`：外部变化信号
- `65_洞察/`：跨资料和项目形成的高价值判断
- `75_提示词库/`：可复用人工智能协作提示词
- `80_复盘/`：周、月、项目复盘
- `85_运行记录/`：自动化巡检报告、运行摘要、前台推送、反馈队列和阶段性指标

## 当前分类规则

- `20_资料库/` 按资料主题或来源场景分层：当前使用 `人工智能产业/`、`人工智能对话/Codex/`、`AI产品与工具/`、`工作流与自动化/`、`设计与视觉/`、`管理与组织/`、`写作与表达/`。
- `30_原子笔记/` 当前保持平铺，不按主题拆目录；等原子笔记数量明显增长后再按领域分层。
- `35_主动回忆/` 当前使用 `回忆卡片/`，后续如有需要再补误区或提问队列。
- `60_行业情报/` 按信号类型分层：`模型与公司/`、`产品发布/`、`开源生态/`、`基础设施与算力/`、`政策与平台/`、`市场与商业化/`。
- `65_洞察/` 按状态分层：`候选洞察/`、`已验证洞察/`、`已应用洞察/`、`已退役洞察/`。
- `75_提示词库/` 按使用场景分层：`收集与录入/`、`萃取与整理/`、`分析与研究/`、`项目推进/`、`写作与表达/`、`前端与视觉/`、`Codex工作流/`。
- `80_复盘/` 按周期或对象分层：`周复盘/`、`月复盘/`、`项目复盘/`。
